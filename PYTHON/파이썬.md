## 변수

- 데이터를 저장하고 처리하는 방법
- 변수 이름 규칙
    - 식별자의 이름은 영문 알파벳, 언더스코어(_), 숫자로 구성
    - 첫 글자에 숫자가 올 수 없음 → 숫자로 시작하면 해당 변수가 숫자가 아니라 변수라는 것을 뒤의 글자까지 봐야 알 수 있기 때문에(컴파일러는 역추적을 지원하지 않음)
    - 길이 제한이 없고, 대소문자를 구별
    - 예약어 사용불가❌자자

## 함수

- 함수를 쓰는 이유
    - Decomposition(분해)
        - 함수를 만들어서 코드를 더 간단히 쓸 수 있음
    - Abstraction(추상화)
        - 내부 구조를 몰라도 사용할 수 있음
- 함수의 구분
    - 내장 함수
        - 파이썬에 기본으로 포함된 함수
    - 외장 함수
        - import문을 통해 사용하며, 외부 라이브러리에 제공되는 함수
    - 사용자 정의 함수
        - 직접 사용자가 만드는 함수
- 직접 변수의 이름으로 특정 Argument를 전달 할 수 있음
    - Keyword Argument 다음에 Positional Argument를 활용할 수 없음❗ (ex. add(x=2, 5) 안됨)
- Default Argument는 항상 마지막에 와야함
- asterisk(*)→ 가변 값을 받을 수 있음
    - ex) a, b, *c = (1, 2, 3, 4, 5)
    - ⇒ a =1 , b= 2, c = [3, 4, 5]
- 가변키워드인자(**) → 키-값 쌍으로 여러개 들어옴 (dict로 묶여서 처리됨)
    - 키는 문자열로 쓰면 안됨!!!
        - ex) print_family_name(dog=’멍멍이’, cat=’냥냥이’)

## 변수의 생명주기

- built-in scope
    - 파이썬이 실행된 이후부터 영원히 유지
- global scope
    - 모듈이 호출된 시점 이후 혹은 인터프리터가 끝날 때까지 유지
- local scope
    - 함수가 호출될 때 생성되고, 함수가 종료될 때까지 유지

## 스코프 범위(LEGB)

- L : local → 현재 작업 중인 영역
- E: Enclosed → 현재 작업 중인 영역 바로 위 영역
- G: Global → 전역 변수, 선언 이후 프로그램 종료시까지
- B: Built-in →프로그램 시작에서 끝까지
- 함수 내에서는 바깥 Scope 변수에 접근 가능하나 수정할 수는 없음
- 함수 안에서 global 식별자를 변수 앞에 붙여주면 바깥 scope의 변수 값을 변경할 수 있음
    - 파라미터는 global을 쓸 수 없다.
    - global 선언하기 전에 변수를 사용할 수 없다.
- nonlocal
    - global을 제외하고 가장 가까운 변수를 찾음
    

## 패키지와 모듈

- 패키지는 pip install로 설치하는 것
- 모듈은 패키지에 속해있는것
- from 패키지 import 모듈
- from 모듈 import (클래스, 메서드 ,변수)
- from 패키지.모듈 import (클래스, 메서드, 변수)
- pip freeze > requirements.txt 를 통해 현재 설치된 패키지 정보 export 가능
- pip install -r requirements.txt 를 통해 패키지 정보 import 가능

## 가상환경

- python -m venv [venv 이름]을 통해 가상환경 만들 수 있음
- pip list 를 통해 설치된 패키지 목록 볼 수 있음
- 가상환경 활성화
    - source venv/Scripts/activate
- 비활성화
    - deactivate
- ❗Range끼리는 더할 수 없다❗
- Range에다가 값을 곱할 수 없다.
